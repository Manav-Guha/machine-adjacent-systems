Analytical Frameworks Explained

How we think about verification, evidence, and the discipline of refusing to overclaim.

Analytical Frameworks Explained, #1

The MAS Thesis: Evidence Over Inference

The AI tooling industry generates output faster than it verifies it. Code without runtime verification. Frameworks without grounding in evidence. Predictions presented as certainties.

Machine Adjacent Systems builds the inverse: verification frameworks where evidence anchors every statement, and the boundaries of what can legitimately be claimed are made explicit.

Two products, one discipline. For AI-generated software (myCode): operational tests with corpus correspondence from 9,297 structured runtime-analysis reports over public repositories. For military innovation assessment (MIAF): structured human scoring with adversarial AI challenge against 28+ validated historical cases.

Different domains. Same epistemic stance: evidence-based declarative statements about behaviour under specific conditions. Not claims. Not predictions.

Analytical Frameworks Explained, #2

Runtime Verification: What myCode Actually Does

AI-generated code arrives without runtime guarantees. The model that produced it had no obligation to verify that the code behaves as intended, handles edge cases, scales under load, or matches the developer's stated purpose.

myCode runs the verification the model didn't.

Submit a repository. Declare what the code should do. myCode executes operational tests against your stated intent. Where the tests fail, the diagnostic report shows where, when, and how. Where structural patterns in your code correspond to patterns from a corpus of 9,297 structured runtime-analysis reports over public repositories that historically encountered failure modes, the corpus model surfaces that correspondence.

This is what we mean by verification: not a guess about what might happen, but observation of what does happen — under declared conditions, against a corpus of what has happened before.

Analytical Frameworks Explained, #3

Corpus Correspondence vs Prediction: The Epistemic Discipline

Machine learning models are increasingly deployed to make predictions about software behaviour — to forecast failures, estimate quality, project performance. The pattern is statistically appealing and epistemically dangerous.

A model trained on historical software cannot legitimately predict what your specific code will do in production. It can identify structural correspondences between your code and patterns observed in the training data. The difference is substantive, not semantic.

myCode's corpus model surfaces structural correspondences. The classifier learns to distinguish failure-prone structural patterns from baseline within its training corpus. What this means in practice: when your code structurally resembles repositories where similar structures historically encountered specific failure modes, the model says so.

What it does not mean: that your specific project will encounter those failure modes. The corpus correspondence is real and measurable. The prediction that often follows from such correspondence is an inferential leap the data does not support.

We make evidence-based declarative statements about your project's current and prospective behaviour under specific conditions. Not claims. Not predictions. The discipline of refusing to overclaim is the differentiation.

Analytical Frameworks Explained, #4

MIAF: Same Discipline, Different Domain

The Military Innovation Assessment Framework applies the same epistemic discipline to a different domain: distinguishing genuine military innovation from technical modernization.

Four analytical registers — Conceptual, Irreversibility, Disruption, Generative — score how an innovation restructures conceptualization, creates path dependencies, alters competitive dynamics, and enables new capabilities. The PPTO stack (People, Process, Technology, Organization) maps where these effects concentrate within institutions.

The framework is human-scored. An adversarial AI agent challenges each scoring decision but is deliberately restricted from providing its own scores. The user adjudicates; the AI argues. Validated against 28+ historical cases spanning 1000 years.

Same discipline as myCode: evidence-grounded discrimination, explicit limits on what the framework claims, human judgement at the centre of adjudication.
